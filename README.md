# 🧩 Shopify — Проведення оптимізації Shopify теми

## 📋 Опис завдання

Мета — закріпити навички створення оптимізованого продукту, покращити показники продуктивності (Lighthouse) та забезпечити правильну роботу адаптивних зображень.

**Завдання:**

1.  **Оптимізація завантаження (Lazy-loading):**
    - Застосувати `loading="lazy"` до всіх зображень нижче першого екрану.
2.  **Адаптивність та Тестування:**
    - Переконатися у коректному відображенні на пристроях з шириною 320px, 768px та 1280px.
3.  **Рефакторинг зображень (Liquid):**
    - Конвертувати застарілий HTML-синтаксис `<img>` у сучасний Liquid-формат використовуючи `image_tag` та `image_url`.
    - Реалізувати підтримку `srcset` та `sizes` для оптимізації під різні екрани (включно з Retina).
4.  **Очищення ресурсів:**
    - Видалити зайві файли з папки `assets` та шаблонів.
5.  **Аудит:**
    - Провести заміри Lighthouse "До" та "Після".

## ✅ Виконана робота

### 🔹 Оптимізація зображень та Liquid Refactoring

- Проведено повний рефакторинг виводу зображень (зокрема у картках товарів та блоках рекомендацій).
- Застарілі конструкції замінено на фільтр `image_tag` у комбінації з `image_url`.
- **Реалізовано логіку `srcset`:**
  - Сформовано масиви `widths` для генерації зображень під різні роздільні здатності.
  - Враховано підтримку **Retina-екранів (x2)**
- **Налаштовано атрибут `sizes`:**
  - Прописані точні медіа-запити (наприклад, `(min-width: 767px) 300px, 250px`), що дозволяє браузеру завантажувати оптимальний за розміром файл, а не найважчий оригінал.

### 🔹 Lazy-loading

- Атрибут `loading="lazy"` додано до всіх зображень, що не входять у viewport при першому завантаженні.

### 🔹 JS адаптація

- Оновлено логіку JavaScript для перемикання варіантів зображень (тамбнейлів).
- Скрипт тепер підміняє не лише `src`, а й `srcset` (`mainImage.srcset = this.dataset.srcset`), що гарантує відображення правильної якості зображення при перемиканні кольорів/варіантів.

## 📝 Примітки

- **Скріншоти** звітів Lighthouse (До/Після) знаходяться у папці `screenshots`.
- **Завдання із зірочкою (Apps):** Встановлення та аналіз впливу сторонніх додатків (wishlist/reviews/analytics) не виконувалося через брак часу(через постійні проблеми зі світлом).
  - _Теоретичний коментар:_ Сторонні додатки зазвичай суттєво знижують показник Performance (особливо TBT - Total Blocking Time) через ін'єкцію важкого JavaScript коду. Для їх оптимізації необхідно використовувати техніки відкладеного завантаження (defer loading) скриптів додатків до моменту взаємодії користувача зі сторінкою.
