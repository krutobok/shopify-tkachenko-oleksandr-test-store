# 🧩 Shopify — Проведення оптимізації Shopify теми

## 📋 Опис завдання

Мета — закріпити навички створення оптимізованого продукту, покращити показники продуктивності (Lighthouse) та забезпечити правильну роботу адаптивних зображень.

**Завдання:**

1.  **Оптимізація завантаження (Lazy-loading):**
    - Застосувати `loading="lazy"` до всіх зображень нижче першого екрану.
2.  **Адаптивність та Тестування:**
    - Переконатися у коректному відображенні на пристроях з шириною 320px, 768px та 1280px.
3.  **Рефакторинг зображень (Liquid):**
    - Конвертувати застарілий HTML-синтаксис `<img>` у сучасний Liquid-формат використовуючи `image_tag` та `image_url`.
    - Реалізувати підтримку `srcset` та `sizes` для оптимізації під різні екрани (включно з Retina).
4.  **Очищення ресурсів:**
    - Видалити зайві файли з папки `assets` та шаблонів.
5.  **Аудит:**
    - Провести заміри Lighthouse "До" та "Після".

---

## ✅ Виконана робота

### 🔹 Оптимізація зображень та Liquid Refactoring

- Проведено повний рефакторинг виводу зображень (зокрема у картках товарів та блоках рекомендацій).
- Застарілі конструкції замінено на фільтр `image_tag` у комбінації з `image_url`.
- **Реалізовано логіку `srcset`:**
  - Сформовано масиви `widths` для генерації зображень під різні роздільні здатності.
  - Враховано підтримку **Retina-екранів (x2)**
- **Налаштовано атрибут `sizes`:**
  - Прописані точні медіа-запити (наприклад, `(min-width: 767px) 300px, 250px`), що дозволяє браузеру завантажувати оптимальний за розміром файл, а не найважчий оригінал.

## 🔧 Технології

- **Shopify Liquid:**
  `image_url`, `image_tag`, `srcset` logic, `capture`
- **HTML5/CSS:**
  `loading="lazy"`, `aspect-ratio`, Responsive Images (`img` w/ `sizes`)
- **JavaScript:**
  Dataset manipulation (`dataset.srcset`), Event Listeners
- **Tools:**
  Chrome DevTools (Network tab with disabled cache for debugging `srcset`), Lighthouse

- **Скріншоти** звітів Lighthouse (До/Після) знаходяться у папці `screenshots`.
- **Завдання із зірочкою (Apps):** Встановлення та аналіз впливу сторонніх додатків (wishlist/reviews/analytics) не виконувалося через брак часу(через постійні проблеми зі світлом).

# ♿ Shopify — Покращення доступності (Accessibility / A11y)

## 📋 Опис завдання

Мета — провести аудит доступності Shopify-теми, виявити порушення стандартів WCAG 2.1 та виправити їх. Основний фокус зроблено на підтримці навігації клавіатурою, коректній роботі скрінрідерів та виправленні семантичних помилок у кастомних секціях.

**Етапи роботи:**

1.  Автоматизований аудит за допомогою **WAVE Evaluation Tool**.
2.  Ручне тестування навігації клавіатурою (`Tab`, `Enter`, `Space`).
3.  Виправлення помилок у HTML/Liquid коді та CSS стилях.
4.  Повторний аудит для підтвердження результатів.

---

## 🔍 Аналіз та Виправлення (Work Done)

Під час аудиту було виявлено та виправлено ряд критичних проблем у секціях `banner-product`, `featured-products`, `designers` та контактній формі.

### 🔹 1. Навігація клавіатурою (Focus Management)

**Проблема:** При ручному тестуванні виявлено, що у секції `banner-product` (та інших елементах) відсутній візуальний індикатор фокусу при переміщенні клавішею `Tab`. Користувач не бачив, де він знаходиться.
**Рішення:**

- Додано класи Tailwind `focus-visible:ring-2! focus-visible:ring-primary!` для інтерактивних елементів.
- Це забезпечило чітке виділення активного елемента без впливу на дизайн при кліку мишкою.

![Проблема з фокусом](screenshots/Проблема%20з%20фокусом.png)
![Виправлення виділення](screenshots/виправлення%20виділення%20по%20табу%20одного%20з%20елементів.png)

### 🔹 2. Семантика та Інтерактивність

**Проблема:** У секції `banner-product` перемикачі кольорів були реалізовані через тег `<img>` з атрибутом `role="button"`. Це ускладнювало взаємодію та не гарантувало повної підтримки клавіатури.
**Рішення:**

- Зображення обгорнуто в нативний тег `<button type="button">`.
- Це автоматично додало підтримку фокусу та подій клавіатури.

### 🔹 3. Форми та Лейбли

**Проблема:** Інструмент WAVE показав помилку "Missing form label" у секції `featured-products` (сортування) та у контактній формі (honeypot).
**Рішення:**

- У селекті сортування знайдено та видалено зайвий пробіл у атрибуті `for` лейбла, що відновило зв'язок з `id` селекта.
- У контактній формі для прихованого поля-пастки (`trap`) додано `aria-label="Spam protection"`.

![Проблема з пропущеним лейблом](screenshots/Проблема%20з%20пропущеним%20лейблом.png)

### 🔹 4. Адаптація для скрінрідерів (ARIA)

**Проблема:** Деякі елементи створювали "шум" або не мали контексту для незрячих користувачів.
**Рішення:**

- **Зірочки рейтингу:** Приховано візуальні символи (`aria-hidden="true"`) та додано загальний опис `aria-label="Рейтинг: 4.5 із 5"`.
- **Кнопки кошика:** Додано динамічний `aria-label="Додати в кошик: {{ product.title }}"`, щоб розрізняти кнопки в списку товарів.
- **Посилання дизайнерів:** Додано `aria-label` з попередженням про відкриття у новій вкладці (`opens in a new tab`).

![Зірочки](screenshots/Зірочки.png)

### 🔹 5. Контрастність (Visual Design)

**Проблема:** У секції `designers` світло-сірий текст та деякі кольорові акценти не відповідали вимогам контрастності WCAG AA.
**Рішення:**

- Замінено кольори тексту на більш темні/контрастні відтінки у CSS.

![Картка дизайнера до змін](screenshots/Картка%20дезайнера%20до%20змін.png)
![Картка дизайнера після змін](screenshots/Картка%20дезайнера%20після%20змін.png)
